2022年、火曜日の朝5時。上海のオフィスで私はGitHub Trendingページを開いた。

AFFiNEの名前がそこにあった——初めて。画面をスクロールして、また戻って、もう一度確認した（信じられなかった）。

その後5ヶ月で、私たちはTrendingに**28回**登場することになる。

## GitHub Trendingとは何か

GitHub Trendingは、**短期間（24時間/週/月）で最も多くスターを獲得したリポジトリ**のランキングだ。

アルゴリズムの詳細は公開されていないが、絶対的なスター数ではなく「獲得速度」が重視される。つまり、スター数が少なくても勢いがあればランクインできる。

## 戦略1：点火タイミングの設計

Trendingに載るには、**1日で数百〜数千のスターを集中的に獲得**する必要がある。

これを私たちは「点火」と呼んでいた。

点火のトリガーになったもの：
- Product Huntローンチ
- Hacker News Show HN
- 大型機能リリース
- 有名エンジニアのシェア

複数チャンネルに**同時に**投稿することで相乗効果が生まれる。

## 戦略2：言語別Trendingを狙う

GitHub Trendingには言語フィルターがある。「All Languages」より特定言語の方が競争が少ない。

AFFiNEはTypeScriptで書かれているため、TypeScript Trendingを優先的に狙った。言語別では**50スター/日でランクインできる**ことも多い。

## 戦略3：連続登場のサイクル

1回Trendingに載ると、それ自体が露出になりさらにスターが増える——好循環だ。

この好循環を維持するために：
- 定期的な機能リリース（月2回以上）
- リリースごとにコミュニティ告知
- 前回のTrending掲載を次の告知に活用

## 戦略4：READMEの最適化

Trendingから訪れたユーザーが最初に見るのはREADMEだ。

ここでの離脱を防ぐために：
- ファーストビューに動くGIF
- 「5分で試せる」クイックスタート
- スター数・コントリビューター数の表示
- 明確なユースケースの提示

README改善後、Trending経由のスター転換率が1.8倍になった。

## まとめ

GitHub Trendingは「偶然」ではなく「設計」できる。

点火タイミング・言語戦略・連続登場サイクル——この組み合わせで、私たちは5ヶ月28回を実現した。

詳細な戦略は [Gingiris Growth Tools](https://gingiris.github.io/growth-tools/) で。
